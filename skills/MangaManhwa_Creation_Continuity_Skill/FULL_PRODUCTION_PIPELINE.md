# Full Manga/Manhwa Production Pipeline

## Pipeline de obra inteira

1. **Series Bible** — premissa, tema, mundo, sistema de poder, estética, público, formato, limites. Saída: documento mestre.
2. **Character Bible** — todos os recorrentes com visual/narrativa/poder/roupa/negativos. Saída: locks prontos.
3. **Continuity Bible base** — estado inicial, cronologia, locais e promessas. Saída: continuidade raiz.
4. **Arc Planning** — dividir obra em arcos com conflito, evolução, revelações e consequências. Saída: mapa de arcos.
5. **Volume/Season Planning** — agrupar capítulos e controlar ritmo macro. Saída: volumes com clímax/gancho.
6. **Chapter Planning** — objetivo, cenas, páginas, viradas e estado final. Saída: capítulo roteirizado.
7. **Scene Map** — espaço físico e emocional por cena. Saída: mapa para storyboard.
8. **Page Script** — função, ritmo, painéis, falas e continuidade por página. Saída: páginas numeradas.
9. **Panel Script** — câmera, ação, pose, expressão, balão e prompt por painel. Saída: storyboard granular.
10. **Bubble Planning** — ordem, rabicho, posição, tipo, texto, áreas proibidas. Saída: letramento planejado.
11. **Image Prompting** — prompt encadeado com locks e negativos. Saída: prompts prontos.
12. **QA Scoring** — pontuar e corrigir. Saída: página/capítulo aprovado ou revisão.
13. **Export State** — resumo para próxima página/capítulo/arco.

## Pipeline por arco
Tema → conflito → antagonista/obstáculo → evolução → revelações → clímax → consequência → gancho.

## Pipeline por volume
Abertura forte → progressão de tensão → midpoint/virada → pré-clímax → clímax → resolução parcial → cliffhanger.

## Pipeline por capítulo
Recap/estado → abertura visual → conflito → escalada → virada → clímax → consequência → gancho.

## Pipeline por página
Entrada de continuidade → objetivo → layout → painéis → balões → prompts → QA → saída de continuidade.

## Pipeline por painel
Função → câmera → personagem → ação → cenário → balão → prompt → risco → correção.

## Pipeline por prompt
Header → Character Lock → Scene Continuity → Composition → Style → Bubble Safe Zone → Negative Prompt.

## Pipeline por QA
Pontuar → identificar falha → corrigir template/roteiro/prompt → reavaliar → exportar.
